# 文档开发规范及流程

## 1、完成文档编写/更新任务

参考 [TDSQL PG内核产品文档框架](https://doc.weixin.qq.com/sheet/e3_ACAARAbdAFw8IUht8GrSAWpQ6gfsj?scode=AJEAIQdfAAoYQa1ViAACAARAbdAFw&tab=BB08J2) 文档编写任务下发至对应编写负责人，接收到编写任务的编写负责人需：

1. 先Fork本项目至本人的工作区，将自己工作区中的项目clone到本地（或者使用工蜂IDE）
    ```shell
    git clone git@git.woa.com:{本人工作区}/tdsql-pg-kernel-doc.git
    ```
2. 或者创建新的分支，若无权限请联系 bartdong 或 hellezhang 协助添加
3. 对您需要完成的编写任务进行相关文档的编写或更新，要求：
    - 在目标时间内完成编写
    - 符合对应文档编写要求
    - 细节注意点：
        - 图片需要放置在对应目录下images文件夹下
        - 由于git文件按字母排序，建议文件名前面增加序号保证文档顺序

4. 完成后提交回您的工作区或新分支
    ```shell
    git add .
    git commit -m "描述您修改了哪些内容"
    git push
    ```
5. 在工蜂平台提交MR及发起评审

## 2、发起代码评审要求

1. 评审人：添加 bartdong 及 hellezhang 作为评审人，选择规则 **“需要指定个数的评审人同意”** ，评审人数目 **“2”**。
2. 必要评审人：发起评审需选择对应模块的 **研发Leader** 作为必要评审人，选择规则 **“需要至少一个必要评审人同意”**。
3. 发起代码评审

## 3、文档评审团评审

1. 文档评审团成员依据文档分类认领/分配评审任务
2. 一周内完成文档评审
    - 文档正确性评审，有需要的话需引入测试团队完成用例
    - 文档易读性评审
3. 文档评审团将修改建议评论至对应MR评论区

## 4、编写负责人优化

1. 编写负责人依据文档评审团修改建议进行优化修改
2. 修改完成后再次联系文档评审团进行评审
3. 文档评审团通过后即可合并至文档库
